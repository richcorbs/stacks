# v0.4.7

## Bug Fixes

- **Fix Kitty graphics image display** — images were showing as binary strings because vterm's APC callback doesn't fire in UTF-8 mode. Replaced with direct APC interception in raw PTY data, stripping sequences before feeding to vterm.
- **Fix image/text overlap** — after stripping an image APC, the cursor is now advanced past the image area so subsequent TUI output flows below it.
- **Fix duplicate images on pane split** — images are now cleared when a terminal resizes, preventing stale images from the old layout.
- **Fix images persisting after process exit** — image state is cleared when the shell process exits and the terminal resets.
- **Fix ⌘K clearing** — replaced Ctrl+L (which echoed `^L`) with Ctrl+U + Ctrl+L for clean screen clear with prompt redraw.

## Improvements

- **Flat icon design** — each tile in the app icon is now a single consistent color with no gradient/shadow. Updated SVG source, icon.png, and AppIcon.icns.
- **Image pruning** — images whose cell areas have been overwritten by text are automatically removed at draw time.
- **Allocator cleanup** — kitty_graphics.zig now uses a configurable allocator (c_allocator) instead of page_allocator for all internal allocations.
- **Dead code removal** — removed ~200 lines of unused code (old vterm APC callback, pre-processor, cursor advance state machine) after the architecture rework.
