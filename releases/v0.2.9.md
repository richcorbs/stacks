# v0.2.9

## Bug Fixes

- **Fixed emoji and wide character positioning** — emoji (e.g. ✅) and CJK characters were rendered at incorrect positions because `drawWideChars` didn't apply the Y-flip coordinate transform used by the main text renderer. Now uses the same `SaveGState → Translate → Scale(1,-1) → draw → RestoreGState` pattern, so wide characters appear in the correct grid cells.
