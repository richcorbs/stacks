# v0.4.0 — Unicode & Emoji

Comprehensive Unicode overhaul. Stacks now correctly renders modern emoji, CJK characters, and complex text sequences.

## Features

- **Flag emoji** — Regional indicator pairs (🇺🇸 🇬🇧 🇯🇵 etc.) render as combined flag glyphs
- **ZWJ sequences** — Family emoji (👨‍👩‍👧‍👦), profession emoji (👩‍💻 👨‍🍳), and rainbow flag (🏳️‍🌈) render as single combined glyphs
- **Skin tone modifiers** — Emoji with skin tones (👋🏻 👋🏽 👋🏿) render correctly
- **Variation selectors** — VS15/VS16 text vs emoji presentation respected

## Bug Fixes

- **Unicode width tables updated from 2007 to 2025** — libvterm's combining and fullwidth tables updated from Unicode 5.0 to 15.1. Fixes grid misalignment with modern emoji, CJK blocks, and symbols.
- **Continuation cells no longer rendered as spaces** — Wide character continuation cells (width=0) were incorrectly clamped to width=1, causing extra spaces after every emoji and CJK character.
- **RawScreenCell struct layout corrected** — The C-to-Zig boundary struct had `i32` for a `char` field; worked by accident on little-endian. Now uses `i8` with correct padding, verified by comptime size check.
- **Line corruption with emoji in terminal output eliminated** — ZWJ and VS16 characters getting their own cells threw off cursor position tracking, causing repeating/growing lines.

## Improvements

- **VTERM_MAX_CHARS_PER_CELL increased from 6 to 10** — Supports complex emoji sequences (family emoji = 7+ codepoints) without truncation.
- **ZWJ glyph scanning in libvterm** — Characters after ZWJ are included in the same glyph at the cell level, so cursor positioning matches program expectations.
- **Visual combining in renderer** — Render-time combining for regional indicator pairs, skin tone modifiers, and ZWJ sequences that can't be combined at the cell level without breaking wcwidth compatibility.
- **Scrollback uses @sizeOf instead of hardcoded struct size** — No longer breaks when cell struct size changes.
