## Bug Fixes

- **Fixed emoji width rendering** — Emoji with variation selector 16 (e.g. ⚠️, ✔️, ☀️) now correctly occupy 2 cells instead of 1, fixing column misalignment in tables and formatted output. The fix handles both inline and split-delivery of the VS16 combining character in libvterm's state machine.

- **Updated Unicode width tables to 15.1** — Replaced libvterm's outdated fullwidth character table with one generated from Unicode 15.1 `EastAsianWidth.txt` (W/F) and `emoji-data.txt` (Emoji_Presentation=Yes). Adds coverage for regional indicators, newer emoji blocks, CJK extensions, and Kana supplements (120 ranges, ~182k codepoints).

## Improvements

- **Update dialog text** — Version info ("you have 0.3.0") now appears on its own line instead of wrapping awkwardly.
