## Bug Fixes

- **Fix terminal not filling window width after Cmd+Enter respawn** — When a terminal command exited and was restarted via Cmd+Enter (sidebar re-activation), the new PTY was spawned with the default 80x24 size. Since the vterm already had the correct dimensions, `syncTermSize` detected no change and skipped the PTY resize, leaving the child process rendering at 80 columns instead of the full window width.

## Improvements

- **PTY spawn now takes explicit dimensions** — `Pty.spawn()` accepts a `WinSize` parameter and passes it directly to `forkpty()`, so the PTY is born at the correct size. This eliminates the bug class where callers forget to resize after spawn.
- **NSSplitView hardening** — Added `setArrangesAllSubviews:YES` and holding priorities to the root split view, ensuring the main panel always fills remaining space after the sidebar.
