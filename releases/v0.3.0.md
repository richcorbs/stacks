# v0.3.0

## Features

- **Split panes stay alive after process exit** — when a shell exits in a multi-pane session, the pane clears and remains in place instead of being auto-removed. The shell respawns when the pane receives focus via ⌘] / ⌘[ or click.
- **Split buttons in header bar** — two new icons to the right of the git status for splitting the focused pane horizontally (⌘D) and vertically (⇧⌘D).
- **Quit confirmation** — ⌘Q now prompts for confirmation when any terminal has a running process.

## Improvements

- **Header layout consolidation** — git status label and split buttons are positioned by a single shared `layoutHeaderRight` function, eliminating layout drift during splits.
- **Layout constants shared** — button dimensions and spacing defined once in `window.zig` and imported by `term_text_view.zig`.
- **Focus border logic extracted** — `shouldShowFocusBorder` helper encapsulates the multi-pane vs single-pane focus border rules.
- **Smarter cwd persistence** — `saveCwdForTerminal` falls back to any live pane's cwd when the focused pane is empty.
- **PTY size sync on respawn** — new shell PTY is resized to match vterm dimensions immediately after spawn, preventing zsh PROMPT_EOL_MARK artifact.
- **Null-safe SF Symbol loading** — split button icons gracefully degrade on older macOS versions.

## Bug Fixes

- **⌘W on empty panes shows confirmation** — previously auto-closed without asking.
- **Git status no longer shifts** during pane splits.

## Tests

- 3 new PTY state transition tests documenting `isClosed`/`hasExited` invariants used by focus borders, confirmation dialogs, and respawn triggers.

## README

- Added speech-to-text feature and Hold Shift shortcut to feature list and keyboard shortcuts table.
